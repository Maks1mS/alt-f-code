NTP Setup

A computer derives its time using a quartz crystal that vibrates at a given frequency.
The crystal frequency depends on the crystal quality, its age and temperature, so it is necessary to calibrate it regularly and compensate its inherent drift and wander.
That is accomplished by using a NTP (Network Time Protocol) server, that keeps its time accurate by synchronizing itself with other higher grade NTP servers.

Usually your ISP will have a NTP server that you should use, or you can instead use a public server based on your country or continent name, such as europe.pool.ntp.org, us.pool.ntp.org, etc.
If the above does not fit, you are advised to use 0.pool.ntp.org, 1.pool.ntp.org, etc, which will use your country nearest public NTP server (and you should not use the generic pool.ntp.org).

At power up, an operating system reads the current time from a battery-powered RTC (Real Time Clock), that itself suffers deviations from the true time. It might thus be appropriate to synchronizes the time at boot time. At poweroff the RTC is updated with the current time.

Under Alt-F you can run a NTP server continuously, which synchronizes itself with another NTP server. The Alt-F NTP server can be used as the source to synchronize other computers in your network, and it continously keeps the current time accurate slowly accelerating or decelerating the clock until the time is correct.
